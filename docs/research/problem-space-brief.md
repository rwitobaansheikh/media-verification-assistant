# Problem Space Brief

Status: Draft

## The Problem

It is becoming harder for people to evaluate whether suspicious media is authentic, synthetic, edited, misrepresented, or missing important context. A user may see an image online and not know where it came from, whether it has been altered, whether it was generated by AI, or whether it is being presented in a misleading way.

This project should help users reason about suspicious media by collecting evidence signals and presenting them in a clear confidence report. One important idea is provenance: understanding where media came from, who created or modified it, and whether there is trustworthy history attached to it.

## Why This Is Hard

There is no single stable signature that proves a piece of media is real or fake. Synthetic media tools change quickly, media is often compressed or re-uploaded, metadata can be missing or stripped, and provenance data is not always available.

Even when a model gives a score, that score is not the same as truth. Detectors can fail on unfamiliar datasets, edited images, screenshots, low-quality uploads, or media from platforms that re-encode files.

## What Signals We Can Use

Possible evidence signals include:

- visual inconsistencies;
- metadata and file information;
- provenance or content credentials where available;
- model-based synthetic media or manipulation detection;
- compression or re-encoding clues;
- source/context notes if the media came from a known URL or post;
- confidence calibration and known limitations.

## What The MVP Should Do

The MVP should focus on a small, realistic slice of the problem, likely image-based media verification. It should accept an image or image-like input, run a limited set of checks, and produce an evidence-backed report that explains what signals were found and how much confidence the system has.

The MVP should be useful as a learning, research, and engineering artifact even if its detection ability is limited.

## What The MVP Should Not Claim

The MVP should not claim to fully solve deepfake detection, misinformation, or media authenticity. It should not say something is definitively real or fake. It should not make moderation, legal, employment, journalistic, or safety-critical decisions for users.

It should support human investigation, not replace it.

## Top Risks

- False positives could wrongly label authentic media as suspicious.
- False negatives could make manipulated media seem trustworthy.
- Missing metadata or provenance could be misinterpreted as evidence of fakery.
- Dataset or model choices may not generalize to real-world media.
- Uploaded media may create privacy and retention risks.
- The team may over-scope the MVP before understanding the domain.
